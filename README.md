# ai-job-vacancy-analyzer

`ai-job-vacancy-analyzer` — CLI-приложение для анализа вакансий под профиль начинающего кандидата, которому интересны офисные, удалённые, back-office, support, QA, data, content, Python и automation-задачи.

Проект помогает:
- оценить, насколько вакансия подходит профилю кандидата;
- увидеть риски: продажи, звонки, размытые условия, физическая нагрузка;
- подготовить вопросы работодателю;
- получить черновик сопроводительного письма;
- сохранить структурированный анализ в CSV.

## Установка на Windows

Откройте PowerShell в папке проекта и выполните:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python src/main.py
```

Альтернативный запуск без активации виртуального окружения:

```powershell
.\.venv\Scripts\python.exe src\main.py
```

## Настройка `.env`

Создайте файл `.env` на основе `.env.example`:

```powershell
copy .env.example .env
```

Для ручных режимов API-ключ не нужен.

Для API-режима заполните `.env`:

```env
OPENAI_API_KEY=ваш_api_ключ
OPENAI_MODEL=gpt-4.1-mini
```

Секретные данные нельзя хранить в `.env.example` или коммитить в репозиторий.

## Запуск CLI

```powershell
python src/main.py
```

После запуска приложение предложит:
1. загрузить пример вакансии из `data/sample_yandex_fintech.txt` или вставить текст вручную;
2. выбрать режим анализа:
   - `manual_prompt` — только вывести готовый промпт для ChatGPT;
   - `manual_json_save` — вывести промпт, принять JSON-ответ от ChatGPT, проверить и сохранить результат;
   - `api` — отправить вакансию в OpenAI API и сохранить структурированный анализ.

## Режим `manual_prompt`

Этот режим нужен, если вы хотите только получить промпт.

Приложение выведет полный промпт на русском языке. Его можно скопировать и вставить в ChatGPT.

Промпт требует от модели вернуть строго валидный JSON без Markdown, пояснений и текста до или после JSON.

В этом режиме CSV-файл не создаётся и не заполняется.

## Режим `manual_json_save`

Этот режим позволяет полностью пройти ручной сценарий без API-ключа:

1. Запустите CLI:

```powershell
python src/main.py
```

2. Выберите вакансию: пример из файла или ручной ввод.

3. Выберите режим `manual_json_save`.

4. Приложение выведет готовый промпт. Скопируйте его в ChatGPT.

5. ChatGPT должен вернуть строго валидный JSON.

6. Вставьте JSON-ответ обратно в CLI.

7. Завершите ввод строкой:

```text
END
```

8. Приложение распарсит JSON, проверит его через Pydantic и сохранит результат в:

```text
output/analyses.csv
```

Если JSON пустой, невалидный или содержит неправильные значения, приложение покажет понятную ошибку и не сохранит данные.

## API-режим

В API-режиме приложение:
1. читает `OPENAI_API_KEY` и `OPENAI_MODEL` из переменных окружения или `.env`;
2. отправляет вакансию в OpenAI API;
3. ожидает JSON-ответ;
4. проверяет результат через Pydantic;
5. только после успешной валидации сохраняет анализ в `output/analyses.csv`.

Если `OPENAI_API_KEY` отсутствует или пустой, приложение не делает API-запрос и переключается в `manual_prompt`.

Если API-запрос не сработал, ответ оказался невалидным JSON или не прошёл Pydantic-валидацию, приложение также переключится в `manual_prompt`.

## Файлы в `output`

После успешного API-анализа или успешного режима `manual_json_save` создаётся:

```text
output/analyses.csv
```

Если файл отсутствует, он создаётся с заголовками. Если файл уже есть, новая строка добавляется в конец без перезаписи старых данных.

Списки `why_it_fits`, `concerns` и `questions_for_employer` сохраняются как JSON-строки с русскими символами.
