# ai-job-vacancy-analyzer

`ai-job-vacancy-analyzer` — это MVP CLI-приложения для анализа вакансий под профиль начинающего кандидата, которому интересны офисные, удалённые, back-office, support, QA, data, content, Python и automation-задачи.

Проект помогает:
- быстро оценить, насколько вакансия подходит профилю кандидата;
- увидеть риски: продажи, звонки, размытые условия, физическая нагрузка;
- подготовить вопросы работодателю;
- получить черновик сопроводительного письма;
- работать даже без API-ключа через готовый промпт для ChatGPT.

## Установка на Windows

Откройте PowerShell в папке проекта и выполните:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python src/main.py
```

Если не хотите активировать виртуальное окружение, можно запускать так:

```powershell
.\.venv\Scripts\python.exe src\main.py
```

## Настройка `.env`

Создайте файл `.env` на основе `.env.example`:

```powershell
copy .env.example .env
```

Для ручного режима API-ключ не нужен.

Для API-режима заполните `.env`:

```env
OPENAI_API_KEY=ваш_api_ключ
OPENAI_MODEL=gpt-4.1-mini
```

Секретные данные нельзя хранить в `.env.example` или коммитить в репозиторий.

## Запуск CLI

```powershell
python src/main.py
```

После запуска приложение предложит:
1. загрузить пример вакансии из `data/sample_yandex_fintech.txt` или вставить текст вручную;
2. выбрать режим анализа:
   - `manual_prompt` — вывести готовый промпт для ChatGPT;
   - `api` — отправить вакансию в OpenAI API и получить структурированный анализ.

## Режим без API-ключа

Если выбрать `manual_prompt`, приложение выведет полный промпт на русском языке. Его можно скопировать и вставить в ChatGPT.

Промпт требует от модели вернуть строго валидный JSON без Markdown, пояснений и текста до или после JSON.

Если выбрать `api`, но `OPENAI_API_KEY` отсутствует или пустой, приложение не будет делать API-запрос. Оно покажет понятное сообщение и автоматически выведет промпт для ручного режима.

В `manual_prompt` режиме CSV-файл не создаётся и не заполняется, потому что это ещё не готовый анализ.

## API-режим

В API-режиме приложение:
1. читает `OPENAI_API_KEY` и `OPENAI_MODEL` из переменных окружения или `.env`;
2. отправляет вакансию в OpenAI API;
3. ожидает JSON-ответ;
4. проверяет результат через Pydantic;
5. только после успешной валидации сохраняет анализ в `output/analyses.csv`.

Если API-запрос не сработал, ответ оказался невалидным JSON или не прошёл Pydantic-валидацию, приложение переключится в `manual_prompt` режим.

## Файлы в `output`

После успешного API-анализа создаётся:

```text
output/analyses.csv
```

В файл записываются структурированные результаты анализа. Списки сохраняются как JSON-строки внутри CSV.
