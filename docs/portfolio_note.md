# Portfolio Note

## Проблема

Начинающему кандидату сложно быстро понять, подходит ли вакансия его профилю. В описаниях часто встречаются размытые обязанности, продажи, звонки, обещания высокого дохода и другие рискованные признаки.

## Решение

Проект помогает разбирать вакансии через CLI. Он сочетает LLM-анализ, ручной JSON-режим без API-ключа, локальную эвристику красных флагов, CSV-хранилище и Markdown-отчёты.

## Что реализовано

- интерактивное меню;
- анализ вакансий из текста и `.txt` файлов;
- режимы `manual_prompt`, `manual_json_save` и `api`;
- fallback при отсутствии API-ключа;
- Pydantic-валидация результата;
- сохранение анализов в CSV;
- просмотр и фильтрация сохранённых анализов;
- экспорт отдельных Markdown-файлов;
- общий Markdown-отчёт;
- локальный risk score с настройкой весов через JSON;
- автотесты.

## Какие технологии использованы

- Python;
- argparse;
- OpenAI SDK;
- Pydantic;
- pytest;
- CSV;
- Markdown;
- JSON config.

## Что можно улучшить

- добавить web-интерфейс;
- добавить импорт нескольких вакансий за раз;
- подключить hh.ru API;
- использовать RAG по резюме кандидата;
- улучшить локальный скоринг и объяснимость флагов.

## Какие навыки демонстрирует проект

- проектирование CLI-приложения;
- работа с внешним API и fallback-сценариями;
- валидация данных через Pydantic;
- файловое хранение данных;
- генерация Markdown-отчётов;
- тестирование бизнес-логики и CLI-команд;
- итеративная разработка по этапам.
